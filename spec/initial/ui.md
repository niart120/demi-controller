# UI設計

## 1. 方針

Project_DemiのUIは、ゲーム画面ではなく入力装置の状態表示と設定に限定する。PySide6のQt Widgetsを使い、入力・設定・終了の操作は標準部品の契約に従う。独自部品はcontroller previewの描画に限定する。

## 2. メインウィンドウ

既定サイズ:

```text
幅 960 px
高さ 640 px
最小 800 x 520 px
```

構成:

```text
┌─────────────────────────────────────────────────────────────┐
│ 接続状態  [接続/切断] [入力開始/停止]  [割当] [接続] [色] │ 52
├─────────────────────────────────────────────────────────────┤
│                                                             │
│              ControllerPreviewWidget                        │
│                                                             │
│        L/ZL                                      R/ZR        │
│             left stick      A/B/X/Y                          │
│             d-pad           right stick                     │
│                                                             │
│             gyro X/Y/Z meters · accel X/Y/Z G                │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ adapter | connection | capture | warning/error              │ 28
└─────────────────────────────────────────────────────────────┘
```

`QMainWindow` の標準領域へ `QToolBar`、中央の `ControllerPreviewWidget`、`QStatusBar` を配置する。中央表示は残り領域へ拡大する。

## 3. ツールバー

ツールバーは `QToolBar` と `QAction` で構成し、独自の座標判定を持たない。

左から次を配置する。

### 3.1 接続状態

表示:

- 未設定
- アダプターなし
- 切断
- 接続中
- 接続済み
- 切断中
- エラー

接続済みかどうかを色だけに依存せず、文字とアイコン形状で区別する。

### 3.2 接続・切断

状態に応じてラベルと処理を変える。

| 状態 | 操作 |
|---|---|
| 未設定 | 接続設定を開く |
| 切断 | 接続 |
| 接続中 | 無効 |
| 接続済み | 切断 |
| エラー | 再試行または設定 |

新規ペアリングは通常の接続ボタンに混ぜず、接続設定内の明示操作にする。

`Ctrl+Enter` はこの状態依存actionと同じ処理を実行する。入力捕捉中も有効とし、接続操作だけを行う場合は捕捉を解除しない。主キーボードの Return とテンキー Enter の両方を受け付ける。

### 3.3 入力開始・停止

- `IDLE` では「入力開始」
- `CAPTURED` では「入力停止」
- 接続中でなくても可視化確認のため開始できる
- 設定ダイアログ表示中は無効
- フォーカスがない場合は無効

未接続で開始した場合、状態バーへ「プレビューのみ。対象機器へは送信していない」と表示する。

### 3.4 設定ボタン

- キー割り当て
- 接続
- コントローラーカラー

ダイアログを開く前にニュートラル化し、排他マウスを解除する。

## 4. ControllerPreviewWidget

### 4.1 図形

公式製品画像を直接使わず、Project_Demi独自の簡略図形を描く。

- 本体: 丸角矩形と左右グリップ
- スティック: 外周円と可動ノブ
- A/B/X/Y: 円
- 十字キー: 4つの丸角矩形
- L/R/ZL/ZR: 上部の台形または丸角矩形
- Plus/Minus/Home/Capture: 小型記号
- IMU: ジャイロ3軸メーターまたは中心レティクルと、加速度3軸のG値

### 4.2 描画方式

`ControllerPreviewWidget.paintEvent()` 内で `QPainter` を使う。入力フレーム受信と再描画要求を分離し、最大60 Hzのタイマーで `update()` を要求する。widget外へ `QPainter`、`QPen`、`QBrush` を保持せず、毎回の描画は最新 `ControllerFrame` と色設定から決定する。pixel完全一致を契約にせず、描画モデル、状態、更新頻度を試験する。

### 4.3 色

設定された4色を両方へ反映する。

- `body`
- `buttons`
- `left_grip`
- `right_grip`

押下状態は元色から十分なコントラストを得られる色を計算する。単純な明度加算だけで白や黒に近い色が判別不能になる場合、輪郭線と内側リングを併用する。

### 4.4 状態表示

`ControllerPreviewWidget` は `ControllerFrame` だけを受け取る。Qt入力状態やswbt状態を直接参照しない。

表示更新:

- ボタン集合: 押下オーバーレイ
- スティック: 中心から半径内へ移動
- ジャイロ: rad/sの数値を表示し、簡易メーターは表示用上限へclampする。raw値はUIへ露出しない
- 加速度: G単位の数値を表示できるが通常モデルの設定項目にはしない。静止時はノルム1、`ACCEL:ZERO` 診断中は完全な0を表示する
- 捕捉: 外枠または上部バナー
- 切断: 半透明の「未接続」表示。ただしプレビューは継続

## 5. キー割り当てダイアログ

`QDialog`、model/view control、`QDialogButtonBox` を使う。binding rowやscroll領域を独自座標で描画しない。

### 5.1 構成

```text
┌──────────────────────────────────────────────┐
│ キー割り当て                     [×]         │
├──────────────────────────────────────────────┤
│ プロファイル [Default ▾]                     │
│                                              │
│ A                 F                [変更]     │
│ B                 V, SPACE         [変更]     │
│ ...                                          │
│ 左スティック上    W                [変更]     │
│ 診断ジャイロY-    I                [変更]     │
│ 診断加速度0G       O                [変更]     │
│ ...                                          │
│ マウスジャイロ    有効             [設定]     │
│                                              │
│ [標準へ戻す]             [取消] [保存]       │
└──────────────────────────────────────────────┘
```

一覧は `QTableView` とmodelで表示し、Qt標準のスクロールを使う。全項目を1画面に詰め込まない。

### 5.2 変更操作

- 「変更」を押す。
- `QDialog` 内へ入力待ち状態と候補を表示し、対象外のcontrolを一時的に無効化する。
- 次のキーまたはマウスボタンを候補表示する。
- ボタン割り当てには「入力を反転」チェックを表示する。スティック方向と診断targetでは無効化する。
- 反転中の割り当ては一覧上でも明示する。
- `Esc` は取消。Esc自体を割り当てる場合は「Escを割り当てる」補助操作を使う。
- 競合がある場合は対象を列挙する。
- 保存前に設定全体を検証する。

### 5.3 ジャイロ設定

同ダイアログ内の `QGroupBox` とform controlで次を編集する。

- 有効
- 水平感度
- 垂直感度
- Y反転
- pitch上限

水平感度と垂直感度は、それぞれ `1.0` を標準とする独立倍率として表示する。一方を変更しても他方の表示値を再計算しない。数値入力には許容範囲と現在値を表示する。

### 5.4 IMU 診断target

`GYRO:Y_NEGATIVE`、`GYRO:Y_POSITIVE`、`GYRO:Z_POSITIVE`、`GYRO:Z_NEGATIVE`、`ACCEL:ZERO` は通常のbinding行として一覧へ表示する。利用者はsourceだけを変更できる。targetの追加・削除、角速度の変更、0G以外の加速度入力は扱わない。

## 6. 接続設定ダイアログ

`QDialog`、`QComboBox`、`QLineEdit`、`QCheckBox`、`QDialogButtonBox` を使う。adapter 0件では接続とpairingを無効にし、再検索を有効に保つ。

項目:

```text
USBアダプター
  [usb:0 ... ▾] [再検索]

コントローラー
  Pro Controller

ボンドスロット
  default

接続タイムアウト
  30 秒

起動時
  [ ] 保存済みボンドで再接続

診断ログ
  INFO

[新規ペアリング] [取消] [保存]
```

`新規ペアリング` は確認画面を挟む。

確認文には次を含める。

- 対象機器側でコントローラー登録画面を開く必要がある。
- 専用USB Bluetoothアダプターが必要である。
- 既存ボンドスロットを上書きする場合の影響。
- 処理中にUSBアダプターを抜かない。

## 7. カラー設定ダイアログ

`QDialog`、`QColorDialog`、`QLineEdit`、`QDialogButtonBox` を使い、色選択と `#RRGGBB` 入力を同期する。

4行の色設定を持つ。

```text
本体          [swatch] #323232
ボタン        [swatch] #0F0F0F
左グリップ    [swatch] #323232
右グリップ    [swatch] #323232
```

0.1.0では大型の色相環を作らず、次で十分とする。

- `#RRGGBB` 文字入力
- 色見本
- `QColorDialog` による色選択
- 既定値へ戻す
- 左右を同じ色にする

接続中に保存した場合:

```text
表示色は更新済みです。
対象機器へ反映するにはコントローラーを再接続します。
[後で] [再接続して反映]
```

## 8. 状態バー

`QStatusBar` と複数の `QLabel` を使い、adapter、connection、capture、warning/errorを分離して表示する。

常時表示する情報:

```text
Adapter: usb:0 | Connected | Input: Captured | 8 ms | warning
```

必要以上に内部プロトコル値を見せない。エラー時は短い要約を表示し、詳細はログ表示またはログファイルへの導線に分ける。

## 9. ダイアログ管理

同時に開ける `QDialog` は1つだけとする。保存と取消には `QDialogButtonBox` の標準buttonを使う。ダイアログを開く処理はApplicationCoordinator経由とし、widgetが設定やruntime状態を直接変更しない。validation errorではダイアログを閉じず、draftと該当controlを保持する。

## 10. レイアウト

`QVBoxLayout`、`QFormLayout`、model/view control、size policyを使い、独自の座標計算やhit testで操作部品を配置しない。中央領域は割合と最小値を使う。

- コントローラー図の最大幅: 中央領域の85%
- 最大高さ: 中央領域の75%
- アスペクト比を保持
- 最小操作部品高さ: 32px
- ツールバーボタン間隔: 8px
- 文字が収まらない場合は省略せず、ウィンドウ最小幅を守る

高DPIではQtの論理寸法とdevice pixel ratioを混同しない。スクリーンショット基準試験はOSごとの差が大きいため、座標と状態の試験を中心にする。

## 11. キーボード操作

最低限の操作:

| キー | UI動作 |
|---|---|
| Tab / Shift+Tab | フォーカス移動 |
| Enter / Space | ボタン実行 |
| Esc | ダイアログ取消 |
| F12 | 入力捕捉解除 |
| Ctrl+Q | 終了 |
| Ctrl+C | 捕捉中のみ捕捉切替 |
| Ctrl+Enter | 接続・切断。主キーとテンキーの両方 |

UIフォーカスとコントローラー割り当ての競合は、入力モードの優先順位で解決する。

## 12. UI受入条件

- 800x520で主要操作が欠けない。
- 設定ダイアログを開いた瞬間にControllerPreviewWidgetがニュートラルを示す。
- 接続処理中もウィンドウ移動と再描画が止まらない。
- 60Hz描画中に入力評価の95パーセンタイル間隔が16ミリ秒を超えないことを診断モードで測定する。
- 色だけを見なくても、接続、捕捉、押下を判別できる。

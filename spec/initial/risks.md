## 2. 検証優先順位

実装前半で確認する:

1. テンプレート生成物と本設計の配置差
2. pyglet 2.1でWindowsの排他マウス差分が安定するか
3. pyglet主スレッドへのワーカーイベント投稿
4. swbt-python 0.2で接続オブジェクトを専用スレッドに閉じ込められるか
5. issue #69/#70の物理単位APIと仮想SPI校正値が同じ尺度を使うか
6. Pro Controllerで水平静止時 `(0, 0, +1) G` が受信側の姿勢解釈と整合するか
7. `ControllerColors` の再生成・再接続で保存済みボンドを利用できるか

実機段階で確認する:

1. 8ミリ秒評価とジャイロ感度
2. 250ミリ秒停止監視の誤発火
3. アダプター抜去時の例外と終了
4. 色設定の対象機器側表示
5. パッケージ版のUSBアクセス

## 3. 変更時の判断基準

- 入力安全性と軽量性が衝突した場合、安全性を優先する。
- UIの部品不足だけを理由にQtへ戻さない。必要部品数と実装量を測って判断する。
- OS差を大量の条件分岐で吸収せず、入力バックエンドまたはplatform adapterへ分離する。
- swbt内部へ依存しないと実現できない機能は0.1.0から外す。
- 確認できていないハードウェア構成を「対応」と表記しない。

## 4. 初期値と調整根拠

| 項目 | 初期値 | 調整根拠 |
|---|---:|---|
| 入力評価 | 8ms | swbtレポート周期との整合、遅延実測 |
| 描画 | 60Hz | CPUと視認性 |
| 停止監視 | 250ms | 誤発火と安全停止 |
| 水平感度 | 1.0 | 独立した無次元倍率。基準値は `π/6000 rad/input unit` |
| 垂直感度 | 1.0 | 独立した無次元倍率。水平感度の比率ではない |
| pitch上限 | ±75° | 極へ到達させず、TPSの仰俯角範囲を確保する |
| 静止加速度 | `(0, 0, +1) G` | Pro Controller基準の+Z面外向き座標と既存リバース資料 |
| ウィンドウ | 960x640 | 設定UIと可視化の収まり |

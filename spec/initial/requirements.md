# 要件

## 1. 前提

この文書の `MUST` は0.1.0で必須、`SHOULD` は0.1.xで満たす、`MAY` は将来候補を示す。

## 2. 機能要件

### FR-001 起動

Project_Demiは `uv run project-demi` および `python -m demi` で同じアプリケーションを起動できなければならない。

受入条件:

- 初回起動で設定ファイルが存在しなくてもウィンドウを表示する。
- Bluetoothアダプターがなくても、可視化と設定画面は利用できる。
- 起動失敗は端末とログへ原因を示し、ボンド情報の内容は出力しない。

### FR-002 アダプター列挙

接続設定画面は、`swbt.list_adapters()` で得られる専用USB Bluetoothアダプター候補を表示しなければならない。

受入条件:

- 「再検索」でUIを停止させずに候補を更新する。
- 候補が0件なら接続操作を無効化し、必要な機材を示す。
- 保存済みIDが見つからない場合、自動的に別アダプターへ接続しない。

### FR-003 接続

ユーザーは保存済みボンドを使う再接続と、新規ペアリングを明示的に選べなければならない。

受入条件:

- アプリ起動だけで新規ペアリングを開始しない。
- 接続中は重複した接続操作を受け付けない。
- 接続失敗時に設定画面へ戻れ、再起動を要求しない。
- 接続完了後、最初にニュートラル状態を適用する。

### FR-004 切断

ユーザーはツールバーから切断できなければならない。

受入条件:

- 切断前にニュートラル送信を試みる。
- 切断処理中は新しい入力フレームを送らない。
- 切断後にマウス捕捉を解除し、保持キーとマウス差分を消去する。

### FR-005 キーボード入力

フォーカス中かつ入力捕捉中のキー押下・解放を、設定されたボタンまたはスティック方向へ変換しなければならない。

受入条件:

- 同じキーの重複押下通知で状態が増殖しない。
- 反対方向を同時に押した軸は0になる。
- 一方を解放した時、残っている反対方向が直ちに有効になる。
- UI入力欄へ文字を入力中はコントローラー入力へ流さない。

### FR-006 マウスボタン入力

マウスボタンを設定されたコントローラーボタンへ変換しなければならない。

受入条件:

- 左、中央、右を区別する。
- 利用可能な環境では追加ボタンを数値名で保存できる。
- 捕捉解除時に全マウスボタンを解放した状態へ戻す。

### FR-007 相対マウス・IMU入力

入力捕捉中はQtの入力イベントとOS固有の相対ポインター入力アダプターを使い、`dx` と `dy` からジャイロ角速度と静的加速度を生成しなければならない。WindowsではRaw Inputによる未加速の相対移動量を優先する。

受入条件:

- カーソルが画面端へ到達しても相対移動を継続できる。
- 1評価周期内の複数イベントを加算し、1回だけ消費する。
- 水平入力をワールド上方向まわりのyawとして扱う。
- 垂直入力をpitchとして扱い、rollを蓄積しない。
- pitchを設定された±上限内へ制限し、上限でもyawを継続する。
- 前回評価からの実経過時間を使ってラジアン毎秒の角速度へ変換する。
- ドメイン座標はPro Controller基準の右手座標系とし、+Xをトリガー方向、+Yを左、+Zをボタン・スティック面から外向きとする。
- 水平姿勢の静止時加速度を `(0, 0, +1) G` とし、現在の仮想pitchに応じて同じ座標系へ投影する。
- yawはワールド上方向まわりの回転なので、静的加速度はyaw角に依存させない。
- raw値への変換はswbt-pythonの公開物理単位APIへ委譲し、Project_Demi独自の換算定数を持たない。
- 次の周期に移動がなければジャイロ角速度を0へ戻すが、静的加速度は現在姿勢に対応する1Gを維持する。
- `dt <= 0` では姿勢を更新せず、ジャイロを0、加速度を更新前姿勢に対応する値とする。
- 水平感度と垂直感度を相互の比率ではない独立倍率として設定でき、Y反転とpitch上限も設定できる。
- 並進加速度、遠心・接線加速度、ノイズ、温度ドリフトは0.1.0では生成しない。

### FR-008 入力捕捉

入力捕捉はユーザーの明示操作で開始・終了しなければならない。

受入条件:

- ツールバーの「入力開始」で捕捉する。
- `F12` で必ず捕捉解除できる。
- フォーカス喪失時に自動解除する。
- 捕捉状態を枠、文字、状態バーのうち2種類以上で表示する。

### FR-009 可視化

ウィンドウ中央にコントローラー状態を表示しなければならない。

受入条件:

- A/B/X/Y、L/R、ZL/ZR、Plus/Minus、Home、Capture、十字キー、左右スティック押下を表示する。
- 押下ボタンを視覚的に区別する。
- 左右スティックの傾きをノブ位置で示す。
- ジャイロ3軸を数値または簡易メーターで示す。
- 表示は実機送信に使うものと同じ `ControllerFrame` を参照する。

### FR-010 キー割り当て

ユーザーはGUIから割り当てを変更し、保存できなければならない。

受入条件:

- 対象操作を選び、「次に押したキーまたはマウスボタン」を取得する。
- 重複割り当てを警告し、ユーザーが確定または取消できる。
- ローカル操作とコントローラー操作の競合を表示する。
- 標準へ戻す操作がある。
- `F12` の捕捉解除は変更できない。

### FR-011 接続設定

接続設定画面は次を扱わなければならない。

- USBアダプターID
- コントローラー種別。0.1.0ではPro Controllerのみ
- ボンドスロット名
- 接続タイムアウト
- 起動時の再接続有無
- 診断ログ水準

### FR-012 コントローラーカラー

ユーザーは本体、ボタン、左グリップ、右グリップの色を `#RRGGBB` で設定できなければならない。

受入条件:

- 入力中にプレビューを更新する。
- 不正な値を保存しない。
- 接続中の変更は「再接続後に実機へ反映」と表示する。
- 再接続操作をユーザーが選べる。
- 設定値を `swbt.ControllerColors` へ境界内で変換する。

### FR-013 設定保存

有効な設定変更はTOMLへ原子的に保存しなければならない。

受入条件:

- 一時ファイルへ書いてから同一ファイルシステム内で置換する。
- 破損ファイルは上書きせず、日時付きで退避する。
- 未知の項目を黙って実行時機能へ反映しない。
- スキーマ版を保持する。

### FR-014 入力反転

ユーザーは任意のボタン割り当てについて、物理入力の押下・解放に対する論理状態を反転できなければならない。

受入条件:

- 各ボタン割り当てに `inverted` を設定できる。
- 通常割り当ては物理入力を押している間だけ対象ボタンを押下する。
- 反転割り当ては物理入力を押していない間だけ対象ボタンを押下する。
- 同じ対象への複数割り当ては、各割り当ての有効判定をORで集約する。
- 反転は特定のキー、マウスボタン、コントローラーボタンへ限定しない。
- スティック方向への割り当てでは反転を使用せず、反対方向のターゲットを指定する。
- 入力捕捉外、フォーカス喪失、設定画面、切断、終了時は反転設定に関係なくニュートラルとする。
- GUIは反転中の割り当てを文字またはアイコンで明示する。

### FR-015 診断

接続状態と失敗原因を調査できる診断情報を残さなければならない。

受入条件:

- タイムスタンプ、OS、Python、Project_Demi、PySide6、Qt、swbt-pythonの版を記録する。
- アダプターIDとドライバー情報は必要範囲に限定する。
- ボンドファイルの内容と秘密値は記録しない。
- 通常ログはローテーションし、無制限に増加させない。

## 3. 非機能要件

### NFR-001 応答性

- 描画目標は60Hzとする。
- 入力状態の評価間隔は8ミリ秒とする。
- 接続、列挙、切断でQtのGUIスレッドを100ミリ秒以上連続して塞がない。
- 設定ファイルI/Oを除く通常UI操作は1フレーム以内に表示へ反映する。

### NFR-002 安全停止

- 捕捉中にUIから250ミリ秒以上フレームが届かなければ、接続ワーカーは物理的な静止状態を表すニュートラルを1回以上適用する。
- ニュートラルIMUはジャイロ0 rad/s、加速度 `(0, 0, +1) G` とし、加速度0Gを定常状態として送らない。
- 未処理例外時はニュートラル、切断、ワーカー停止の順で最善努力する。
- `os._exit` を通常終了経路で使用しない。

### NFR-003 可搬性

- OS固有処理を `demi.platform` または具象アダプターへ隔離する。
- パス区切り、設定保存先、フォント名を固定しない。
- Windows、macOS、Linux用ビルドを別々に作る。別OS向けの交差ビルドを前提にしない。

### NFR-004 軽量性

- GUIはPySide6のQt Widgetsを使用し、ElectronとWebViewを0.1系へ導入しない。
- GUIの実行時依存はPySide6を中心とし、QtのGUI型をdomain、application、controller、configへ持ち込まない。
- コントローラー画像は必要な解像度へ限定し、動画や大規模UI素材を含めない。
- 常駐中に毎フレーム新しい大量オブジェクトを作らない。

### NFR-005 試験可能性

- キー・マウスから `ControllerFrame` までをBluetooth機材なしで試験できる。
- 接続処理は偽 `ControllerPort` へ差し替えられる。
- 実時間を直接参照せず、時計インターフェースを注入できる箇所を設ける。
- 実機試験を通常の単体・統合試験から分離する。

### NFR-006 保守性

- 公開関数と公開クラスに型注釈を付ける。
- ドメイン層の分岐は列挙型と不変値オブジェクトで表す。
- 設定スキーマ変更には移行処理と旧版fixtureを追加する。
- 100文字の行長を基準とする。

### NFR-007 ローカル性

- 0.1.0はネットワーク通信を行わない。
- 利用統計、クラッシュ報告、更新確認を自動送信しない。
- 設定、ボンド、ログはユーザーのローカル領域へ保存する。

## 4. 受入シナリオ

### AC-01 通常再接続

1. 保存済みボンドと有効なUSBアダプターがある。
2. アプリを起動する。
3. 「接続」を押す。
4. 接続済み表示になる。
5. 入力捕捉を開始する。
6. `F` を押す。
7. UI上のAが点灯し、対象機器でA入力になる。
8. `F` を離す。
9. UIと対象機器でAが解除される。

### AC-02 フォーカス喪失

1. 接続・捕捉中に `W` と左マウスボタンを保持する。
2. Alt+Tabなどで別ウィンドウへ移る。
3. 250ミリ秒以内に入力捕捉が解除される。
4. 左スティックとZRがニュートラルになる。
5. Project_Demiへ戻っても自動捕捉しない。

### AC-03 UI停止監視

1. 接続・捕捉中に入力フレーム供給を試験用に停止する。
2. 250ミリ秒を超える。
3. 接続ワーカーがニュートラルを適用する。
4. UIへ「入力監視タイムアウト」を通知する。
5. フレーム供給再開後、明示的な再捕捉で操作を再開できる。

### AC-04 設定破損

1. `settings.toml` を不正な構文にする。
2. アプリを起動する。
3. 破損ファイルが退避される。
4. 安全な初期設定で起動する。
5. UIへ設定復旧メッセージを一度表示する。

### AC-05 色変更

1. 接続中に本体色を変更する。
2. UIプレビューが即時に変わる。
3. 実機反映には再接続が必要と表示される。
4. 「再接続して反映」を選ぶ。
5. ニュートラル、切断、再生成、再接続の順で処理する。
